Dingo's git source code repository is available on Github. The latest version of the source is called master, which is very unstable and consistently changing. It may contain lots of unfixed bugs and potential threats to your data. If you can't handle these risks please use our latest official release.

{# Install Dependencies #}

1. Install Dependencies

Ubuntu

Fedora

Others

{# Install on Ubuntu #}

a. On Ubuntu

You will need to run the following commands to get all the build dependencies in Ubuntu: 

      $ sudo apt-get install git autoconf automake libtool intltool gettext g++ make 
      $ sudo apt-get build-dep dingo
{# Install on Fedora #}

b. On Fedora

You will need to run the following commands to get all the build dependencies in Fedora: 

      $ sudo yum install git autoconf automake libtool intltool gettext gcc-c++ make
      $ sudo yum-builddep dingo
{# Install on Other Distros #}

c. On Other Distros

With distros other than Ubuntu and Fedora, you will need to install the following dependencies:

gtkmm(-dev) >= 3.4.0 libnotify(-dev) >= 0.7.5
gstreamermm(-dev) >= 0.10.10 boost-program-options(-dev) >= 1.48
libmediainfo(-dev) >= 0.7.59 libxml++(-dev) >= 2.34.2
sqlite(-dev) >= 3.7.11
intltool >= 0.50.2 gettext >= 0.18.1.1
git >= 2.34.2 autoconf >= 2.34.2
automake >= 2.34.2 libtool >= 2.34.2
make >= 2.34.2 g++ >= 4.7.2

{# Get the Latest Source Code #}

2. Get the Latest Source Code

After installing all necessary dependencies, you can use git to check out a copy of Dingo's latest development version: 

      $ git clone git://github.com/phongvcao/dingo.git
{# Build, Install and Run #}

3. Build, Install and Run

Finally, you just need to move to the folder where you checked the source code out from Github, build the necessary components, then install into your system: 

      $ cd /path/to/dingo/directory
      $ ./configure
      $ make
      $ make install
