{# Installation #}

Installation

1. Why isn't Dingo installation package available on my platform?

At the moment due to the small amount of maintainers and contributors we only support installation package on the three most popular Linux distribution: Ubuntu, Fedora and Linux Mint (which is based upon Ubuntu and can use Ubuntu's debs).


If Dingo installation package is not yet available on your system you can either install Dingo from source, send a package-building request email to our mailing-list or build an installation package for your own system from our latest source code.


2. What are the required libraries to build Dingo?

The libraries required to build Dingo along with their versions can also be found on our Download page - Development section. Here are what you need to build the currently latest stable release:

gtkmm(-dev) >= 3.4.0 libnotify(-dev) >= 0.7.5
gstreamermm(-dev) >= 0.10.10 boost-program-options(-dev) >= 1.48
libmediainfo(-dev) >= 0.7.59 libxml++(-dev) >= 2.34.2
sqlite(-dev) >= 3.7.11
intltool >= 0.50.2 gettext >= 0.18.1.1
git >= 2.34.2 autoconf >= 2.34.2
automake >= 2.34.2 libtool >= 2.34.2
make >= 2.34.2 g++ >= 4.7.2


3. Where can I find libmediainfo?

If you are using Ubuntu or any Ubuntu-based distro such as Linux Mint, libmediainfo should be available on your Software Center. If not you can always add the Official MediaInfo PPA into your Software Sources. OMG! Ubuntu has an excellent tutorial about How to add a PPA to Software Sources in Ubuntu. Things might look a little different in your system but they should be somewhat similar.


With Fedora we distribute libmediainfo along with the installation package, but it will only be the minimum version required to build that particular release of Dingo. If you want a newer libmediainfo, you can always download the latest RPMs on MediaInfo Homepage.


For all other systems we recommend checking out MediaInfo Homepage for your libmediainfo installation.


4. Can Dingo be installed on Windows and Mac OSX?

At the moment the answer is No due to the fact that some Dingo's dependencies (such as gtkmm and gstreamermm) only support Linux. Please see the About page for more information.

{# Features #}

Features

1. How and where does Dingo get its cover art?

Dingo gets the cover art from the audio/video file itself, then stores the cover art as cache. Later on when the file is played again the cover art will be fetched from the cache, which improves cover art reading speed.


2. Does Dingo support audio/video format X (mp3, aac, mpeg ,etc)?

Dingo is developed using GStreamermm, the official C++ binding for GStreamer. Therefore, Dingo can play any file format supported by the GStreamer codecs you have installed on your system.


3. I can't play my audio and/or video files. What is going wrong?

If you cannot play your audio and/or video files, please make sure that the files are accessible by non-root users and that the partitions containing them are properly mounted and accessible.


After the above step if your media files are still unplayable it might be because your system lacks the necessary codecs, and you will have to install extra GStreamer-plugins for them to work.


If the two methods above do not work please open a bug in our Github issues tracker. We will take a look at it and get back to you as soon as possible.


4. I can play audios, but with videos only the sound is played and the images are not shown. What did I do wrong?

If your videos do not show pictures, it is mostly because your system lacks the video codecs required to play the files. Please install gstreamer-plugins-bad, gstreamer-plugins-ugly and gstreamer-ffmpeg, then restart Dingo.


If the above step does not solve the problem please open a bug in our Github issues tracker. We will take a look at it and get back to you as soon as possible.

{# Technical #}

Technical

1. Where does Dingo store its configuration/database files?

They are stored under ~/.config/dingo-0.5/. The default settings file (dingo-default.dgs) and the database file (dingo.db) are stored under two different subdirectories called settings/ and database/, respectively.


The settings file is in XML-based format, while the database file is a SQLite3 database.


2. Where does Dingo store its cache files?

Dingo's cache files are basically cover-arts of all the played audios/videos. They are stored under ~/.cache/dingo/cover-art.
