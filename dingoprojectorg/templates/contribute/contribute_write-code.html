Getting Dingo

Before getting started writing code for Dingo make sure you have our latest development version.

{# Building Dingo #}

Building Dingo

Once you have got the latest development version of Dingo with all the necessary dependencies, it is very easy to build the development package. We use the GNU standard build system, Autotools, for Dingo. Once you have satisfied all the required dependencies you are ready to build: 

      $ cd dingo
      $ ./configure
      $ make
      $ make install

In case you are not very familiar with Autotools we recommend checking out Autotools: A Practioner's Guide to GNU Autoconf, Automake, and Libtool and The Official Automake Documentation.

{# Adding Features #}

Adding Features

Dingo began as a small personal project of its author and our aim is to provide a simple yet effective way for Linux users to manage and enjoy their audios/videos collections. Thus, we think really hard when it comes to new feature: Will it be beneficial to a large amount of users? Will it conflict or duplicate with other functions of the software? etc.. If you are going to extend Dingo we strongly recommend reading Havoc Pennington's excellent article on this matter before getting started.

{# Hacking & Patching #}

Hacking & Patching

We use Git to manage Dingo's source code. To produce a patch, please first commit to your local git branch, then produce the patch using `git format-patch HEAD~1` to include your commit message and author information.


After finishing the patch and commiting it, please create an issue in Dingo's Github issues tracker and label it "enhancement". You should describe in as much details as possible about the patch, the severity, and whether (in your opinion) the patch is a good or bad solution for the problem.


At the time this page is written, Github does not yet have a way to attach files into bug reports. Therefore, we recommend that you create a Github account, then create a repo named dingo-PATCH_NAME and push your patch up there. Or you can compress the patch and upload it somewhere and insert the download link into the report. The report will be reviewed and if it is approved the maintainer will commit the patch.


Please do not send your patch report to our mailing-list, because Dingo's developers automatically get emailed when you open an issue in Github.
